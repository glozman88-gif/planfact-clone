# Техническая спецификация поведения сервиса ПланФакт (для построения клона)

## 1. Модель данных

### 1.1. Юрлицо (LegalEntity)
- Поля: `название`, признак `моё юрлицо / контрагент` (переключатель «Это моё юрлицо ↔ Преобразовать в контрагента»).
- Каждый счёт привязан к юрлицу. Операции и остатки агрегируются по юрлицам.
- Переводы между собственными юрлицами не должны попадать в доходы/расходы (только Перемещение).
- Фильтрация по юрлицу доступна в отчётах (ДДС, ОПиУ, Баланс), включая Баланс по отдельному юрлицу.

### 1.2. Счёт (Account)
- Поля: `название`, `юрлицо` (обяз.), `тип`, `валюта` (обяз.), `группа` (опц.), `реквизиты` (для безнал/электронных), `комментарий`, `начальный остаток` + `дата начального остатка`.
- **Типы счёта:** наличный / безналичный / карта физлица / электронный. Дополнительно по назначению: расчётный, депозитный/овернайт, кредитный (овердрафт), кредитная карта (= счёт с начальным остатком = кредитному лимиту), подотчётный счёт сотрудника.
- **Критическое правило:** дата начального остатка ОБЯЗАНА быть строго раньше даты первой операции по счёту, иначе остатки считаются неверно.
- Счёт — это разновидность актива с движением денег; по нему идут Поступления/Выплаты, формирующие ДДС.
- Группировка в группы; виды отображения «По группам» / «По юрлицам». Заголовок (шапка) показывает общий остаток по всем счетам.

### 1.3. Контрагент (Counterparty)
- Поля: `название`, `группа`, `статья по умолчанию для поступлений`, `статья по умолчанию для выплат`, флаг `Применять статьи по умолчанию`.
- Тип: клиент / поставщик / сотрудник / физлицо. Создаётся автоматически из выписки, вручную, из формы операции.
- При включённой опции «Применять статьи по умолчанию» всем операциям контрагента автоматически проставляется его статья. Отключать, если платежи идут по разным статьям.
- **Приоритет:** правило импорта важнее статьи по умолчанию из карточки контрагента.
- По контрагенту/группе считается дебиторская и кредиторская задолженность (сальдо взаиморасчётов из разрыва дат). Поддержка группировки, объединения, архивирования, экспорта.

### 1.4. Учётная статья (Article)
Центральная сущность классификации. Иерархическая (неограниченная вложенность; родитель имеет собственную сумму + агрегат подстатей). Нельзя создать две статьи с одинаковым именем.

**Пять типов (разделов) статей и их попадание в отчёты:**

| Тип статьи | Подразделы | ОПиУ | ДДС | Баланс |
|---|---|---|---|---|
| **Доходы** | — | ✓ | ✓ | — |
| **Расходы** | — | ✓ | ✓ | — |
| **Активы** | Оборотные / Внеоборотные | — | ✓ | ✓ |
| **Обязательства** | Краткосрочные / Долгосрочные | — | ✓ | ✓ |
| **Капитал** | (в т.ч. «Прочие статьи капитала», «Корректировка», «Дивиденды») | — | ✓ | ✓ |

- ОПиУ строится ТОЛЬКО из Доходов и Расходов. ДДС — из всех пяти разделов (любая денежная операция). Баланс — из Активов/Обязательств/Капитала + нераспределённая прибыль из ОПиУ.

**Активные vs пассивные статьи** (определяют знак в Начислениях):
- **Активные** (растут по дебету): все Активы, все Расходы, статья «Дивиденды» (в Капитале).
- **Пассивные** (растут по кредиту): все Обязательства, все Доходы, Капитал (кроме Дивидендов).

**Дополнительные поля статьи:**
- **Вид денежного потока** (для ДДС): Операционный (О) / Инвестиционный (И) / Финансовый (Ф) — наследуется операцией.
- **Правило валютной переоценки:** по курсу на дату операции (исторический) / на дату построения баланса (текущий).

**Классификация расходов** (выбирается глобально в Общих настройках, влияет на структуру ОПиУ): «прямые/косвенные» | «постоянные/переменные» | без классификации. Прямые ≠ переменные (разные принципы: переменные зависят от объёма, прямые относятся к конкретному проекту/контракту).

### 1.5. Проект (Project) и Группа проектов
- Проект: `название`, `комментарий`, `группа` (опц.), `статус`, флаг `архивный`, `бюджет проекта` (опц., источник плана).
- Группа проектов: `название`, `комментарий`. Ровно два уровня: группа → проекты.
- Проект — аналитический разрез поверх операций разных типов; не отдельный счёт.
- **Статусы (правила):** Плановый (нет операций или все плановые — авто), В работе (есть хотя бы одна фактическая операция — авто), Завершён (ставится вручную).

### 1.6. Товар/Услуга (Product) — см. §9.3.

### 1.7. Сделка (Deal) — см. §7.

### 1.8. Операция (Operation) — см. §2.

### 1.9. Статусы/состояния операции
- **Статус оплаты:** подтверждена / не подтверждена (план vs факт по кассе).
- **Статус начисления:** подтверждено / не подтверждено (план vs факт по начислению).
- **Просрочена** (знак «!»): хотя бы одна дата наступила, но не подтверждена.
- Статусы импорта (при сверке): Новая / Существует / Изменена / Нет в выписке.

## 2. Операции

### 2.1. Базовый принцип
Тип операции определяется ИСКЛЮЧИТЕЛЬНО направлением движения денег, а не движением товара (продажа товара = Поступление, хотя товар убывает).

### 2.2. Типы операций
1. **Поступление** — деньги на счёт (+). ДДС: приток. ОПиУ: доход (если статья — доходная).
2. **Выплата** — деньги со счёта (−). ДДС: отток. ОПиУ: расход (если статья — расходная).
3. **Перемещение** — между своими счетами. Два блока:
   - Списание: дата, счёт-источник + юрлицо, сумма списания.
   - Зачисление: дата зачисления, счёт-получатель + юрлицо, сумма зачисления (отдельная сумма только если валюты различаются → курсовая разница).
   - НЕ влияет на ОПиУ. В ДДС нейтрально (− на одном, + на другом). Можно разбить на Выплату+Поступление и обратно.
4. **Начисление** — проводка между двумя учётными статьями без движения денег. Два блока (Дебет / Кредит). НЕ влияет на ДДС. Влияет на ОПиУ (если затронута статья дохода/расхода) и на Баланс.
5. **Отгрузка** (только в Продажах) — исполнение обязательства поставить товар/услугу клиенту.
6. **Поставка** (только в Закупках) — получение товара/услуги от поставщика.

### 2.3. Поля операции Поступление/Выплата
- Обязательные: `дата оплаты`, флаг `подтвердить оплату`, `счёт` + `юрлицо`, `сумма` (в валюте счёта).
- Дополнительные: `дата начисления` + флаг `подтвердить начисление`, `контрагент` (необязателен), `статья`, `проект`, `сделка`, `назначение платежа` (обязательность настраивается), файлы, комментарии.
- По умолчанию поле «Дата начисления» скрыто и = дате оплаты; включается в Общих настройках.
- **Привязка к сделке блокирует редактирование даты начисления** (берётся из логики сделки/отгрузки).

### 2.4. Две даты (ядро всей логики)
- **Дата оплаты** — фактическое движение денег → драйвер ДДС и остатков счетов (кассовый метод).
- **Дата начисления** — момент возникновения дохода/расхода (акт, отгрузка, оказание услуги) → драйвер ОПиУ по методу начисления и дебиторки/кредиторки.
- Если дата начисления не указана → она = дате оплаты.
- На остаток счёта влияют ТОЛЬКО операции с подтверждённой датой оплаты. В ОПиУ (начисление) попадают только операции с подтверждённой датой начисления.

### 2.5. Операция «Начисление»: правило знаков (двойная запись)

| Позиция | Активная статья | Пассивная статья |
|---|---|---|
| **Дебет** | сумма ↑ (зачисляется) | сумма ↓ (списывается) |
| **Кредит** | сумма ↓ (списывается) | сумма ↑ (зачисляется) |

Поля: дата начисления, юрлицо, статья по дебету, статья по кредиту, сумма, валюта, назначение. Примеры:
- Списание себестоимости: Дт Себестоимость (Расход↑) / Кт Запасы (Актив↓).
- Оприходование излишка: Дт Запасы (Актив↑) / Кт Излишек (Доход↑).
- Каждая операция Начисление сбалансирована (Дт = Кт), поэтому равенство Активы = Пассивы сохраняется автоматически.

### 2.6. Разбиение операции на части («Разбить сумму»)
- Измерения разбивки: дата начисления, контрагент, статья, проект, сделка продажи, сделка закупки. Допускается многомерная разбивка.
- UI: по умолчанию 2 строки, «Добавить…» добавляет, X удаляет, кнопка в шапке колонки «Сумма» делит поровну.
- **Сумма должна быть распределена полностью**; нераспределённый остаток допустим только как явно пустое поле «Не выбрано».
- «Отменить разбиение» теряет данные частей.
- Отображение: операция сворачивается/разворачивается; при фильтре по части показывается вся операция, несовпавшие части блёкло.
- **Групповые/массовые действия с разбитыми операциями недоступны.**
- Разбивка по дате начисления = ручное размазывание признания по периодам при единой оплате.

### 2.7. Распределение на период (автоматическое разбиение по начислению)
- Назначение: равномерное признание расхода/дохода будущих периодов (страховка, лицензии, аренда, подписки) в ОПиУ при разовой оплате.
- Единицы: дни, недели, месяцы, кварталы, полугодия, годы (1–12 месяцев для импорта-правил).
- Операция автоматически делится на части = числу месяцев в интервале.
- **Сумма каждой части ∝ числу дней этого месяца в интервале.**
- Дата начисления части = последний день соответствующего месяца (последняя часть — конечная дата периода).
- ДДС не дробится (платёж остаётся на дате оплаты).
- Минимумы интервала: дни/недели — должны заходить минимум на 1 день/неделю в следующий месяц (≈21 день / 3 недели от середины месяца); месяцы (с 1-го числа) — минимум 2; кварталы/полугодия/годы — без минимума.
- Пример: 1500 ₽, начисление 01.04, 45 дней → 30.04: 1000 ₽ (30 дн), 15.05: 500 ₽ (15 дн).
- Пример: 2500 ₽, начисление 24.09, 2 мес → 30.09: 286,89 ₽ (7 дн), 31.10: 1270,49 ₽ (31 дн), 23.11: 942,62 ₽ (23 дн).

### 2.8. Повторяющиеся операции
- Опция «Повторить»: интервал (дни/недели/месяцы/кварталы/полугодия/годы) + число повторов.
- При «Создать» исходная и ВСЕ повторы создаются одновременно как набор отдельных операций.

### 2.9. Подтверждённость (план/факт на уровне операции)
- Каждая из двух дат подтверждается независимо.
- Будущую дату подтвердить нельзя (только наступившую).
- **Фактическая операция** — с подтверждённой датой; **плановая** — с неподтверждённой.
- Факт = Σ подтверждённых; План = Σ подтверждённых + Σ запланированных.

### 2.10. Прочие действия
- «Создать ещё одну операцию» (форма остаётся открытой), Копирование (дата может перебиться на текущую по настройке), Редактирование, Удаление (с подтверждением).
- **Undo отсутствует** — удаление и групповые правки необратимы.
- Групповые действия: выделение (чекбоксы / Shift-диапазон); массово — смена дат/счёта/суммы/контрагента, подтверждение/снятие дат, конвертация Выплаты в Перемещение, экспорт, удаление. Поля с разными значениями → «Оставить как есть».
- Файлы: ≤5 МБ/файл, до 10 файлов; комментариев без ограничения. Видны всем; редактировать/удалять — автор или владелец аккаунта.
- Итоги внизу раздела «Операции»: число и сумма по типам, разница Поступления − Выплаты. **Нюанс:** «Итого» по счёту НЕ включает начальный остаток.

### 2.11. Влияние на отчёты (сводно)
- **ДДС:** только кассовый, дата оплаты, только подтверждённые. Поступление +, Выплата −, Перемещение нейтрально, Начисление не влияет.
- **ОПиУ:** кассовый или начисление; дата начисления (или оплаты); только статьи Доходов/Расходов с подтверждённым начислением; Перемещения не входят; Начисления входят, если затронута статья дохода/расхода.
- **Баланс:** остатки Активов/Обязательств/Капитала; дебиторка/кредиторка из разрыва дат; Начисления двигают балансовые статьи.

## 3. Отчёт ДДС (движение денежных средств, Форма №4)

### 3.1. Метод и формула
- **Строго кассовый метод**, по дате оплаты. Только подтверждённые оплаты.
- **Формула:** `Остаток на конец периода = Остаток на начало + Денежный поток за период`, где Денежный поток = Поступления − Выплаты по всем потокам.

### 3.2. Структура — три потока (по виду деятельности статьи)
- **Операционный (О):** регулярные платежи основной деятельности (материалы, зарплата, реклама, выручка), запасы, залоговые платежи, краткосрочные обязательства/займы до 1 года.
- **Инвестиционный (И):** нерегулярные платежи по средствам производства со сроком >1 года — покупка/обслуживание/продажа ОС и НМА; финвложения/депозиты.
- **Финансовый (Ф):** привлечение/возврат финансирования — кредиты, займы, овердрафт, дивиденды, взносы учредителей, долгосрочные обязательства.
- Принцип: поступления одного потока служат источником выплат другого.

### 3.3. Правила суммирования и остатки
- Перемещения между своими счетами взаимокомпенсируются (чистый поток = 0).
- Начисления НЕ попадают в ДДС.
- Контроль остатков: способ «По периодам» → «По дням» → остаток по счёту на конец дня; блок «Остатки на конец периода».
- При отсутствии авансов/отсрочек ДДС по операционной деятельности совпадает с ОПиУ (кассовый).

### 3.4. Способы построения и UI
- Способы: По периодам (год/квартал/месяц/неделя/день; разрезы счёт/юрлицо/контрагент/проект), По проектам, По сделкам. Везде разбивка по статьям.
- Суммы кликабельны → список операций → переход в раздел Операции. Колонка «Тренд». Экспорт в Excel. Подсказка о нераспределённых операциях с кнопкой «Распределить». Мультивалютность (валюта аккаунта, конвертация по ЦБ РФ на дату операции).

## 4. Отчёт ОПиУ (прибыли и убытки, P&L / ОДР / Форма №2)

### 4.1. Методы и базовая формула
- Поддерживает **оба метода** (переключатель). По умолчанию — метод начисления.
- **Кассовый:** доход/расход по дате оплаты. **Начисления:** по дате начисления.
- **Базовая формула:** Прибыль = Доходы − Расходы (только статьи разделов Доходы/Расходы).
- Один и тот же набор операций даёт разные цифры в ДДС (всегда касса) и ОПиУ (выбранный метод) из-за разных дат.

### 4.2. Структура (уровни прибыли)
Рекомендуемая последовательность расчёта (формула показывается при наведении):
1. **Выручка** (Σ доходных статей основной деятельности).
2. **Прямые расходы / Себестоимость** (формируется при отгрузке через Начисление Дт Себестоимость / Кт Запасы; зависит от классификации расходов).
3. **Валовая прибыль** = Выручка − Прямые расходы.
4. **Косвенные (операционные) расходы**.
5. **Операционная прибыль** = Валовая − Косвенные.
6. **EBITDA** (промежуточный уровень до амортизации/процентов/налогов).
7. **Амортизация** (Начисление Дт Амортизация(Расход) / Кт ОС(Актив), помесячно, линейно, вручную; неденежная — в ДДС не попадает).
8. **Проценты** (по кредитам/займам/овердрафту/лизингу — отдельные расходные статьи).
9. **Налоги**.
10. **Чистая прибыль** = результат после всех расходов.
11. **Дивиденды** (статья Капитала, активная).
12. **Нераспределённая прибыль** = Чистая прибыль − Дивиденды + Курсовые разницы (перетекает в Капитал Баланса).

### 4.3. Способы построения и UI
- По периодам / По проектам (разница доходов−расходов проекта) / По сделкам (рентабельность сделки). Разбивка по статьям.
- Прибыль по сегментам/направлениям/проектам. Структура ОПиУ зависит от глобальной классификации расходов (прямые/косвенные | постоянные/переменные | нет).
- Мультивалюта: метод начисления пересчитывает по курсу ЦБ на дату начисления. Кликабельные суммы, тренды, экспорт в Excel.

## 5. Баланс (Balance Sheet)

### 5.1. Главное равенство
**Активы = Пассивы**, где Пассивы = Обязательства + Капитал. Всегда сбалансировано (за счёт двойной записи Начислений).

### 5.2. Структура

**АКТИВЫ**
- *Оборотные:* Денежные средства (остатки счетов на дату), Дебиторская задолженность, Запасы (товары/сырьё/готовая продукция), Залоговые платежи, Выданные займы до 1 года, Финансовые вложения (краткосрочные), Отправленные платежи (внутренние переводы), Переводы в пути (служебная — при разных датах списания/зачисления), Факторинг, прочие («Нам должны вернуть», «Взаимозачёт» если нетто-кредитор, Вх. НДС).
- *Внеоборотные:* Основные средства (нетто = за вычетом накопленной амортизации), Выданные займы от 1 года, Финансовые вложения (депозиты/ценные бумаги), Нематериальные активы (ИС, ПО, торговые марки).

**ПАССИВЫ → Обязательства**
- *Краткосрочные:* Кредиторская задолженность, Платежи третьим лицам (разница транзита), Полученные займы до 1 года, Полученные платежи (внутренние от других юрлиц), Овердрафт, НДС к уплате (Исх. НДС), Долг перед сотрудником, прочие («Мы должны вернуть»).
- *Долгосрочные:* Кредиты (остаток тела долга), Полученные займы от 1 года, Лизинг.

**ПАССИВЫ → Капитал**
- Уставный капитал, Вложения учредителей (безвозвратные), Нераспределённая прибыль (= чистая прибыль − дивиденды + курсовые разницы), Другие статьи капитала / Корректировка (служебная для ввода остатков).

### 5.3. Механизм формирования
- **Накопительно с начала ведения учёта по дату построения включительно** (НЕ за период, в отличие от ДДС/ОПиУ).
- Денежные средства = начальные остатки счетов + все операции по дату.
- Дебиторка/кредиторка возникают из разрыва даты начисления и даты платежа (метод начисления): начислили доход, не получили оплату → дебиторка; начислили расход, не оплатили → кредиторка.
- Чистая прибыль из ОПиУ → строка Нераспределённая прибыль (Капитал). Начальные остатки счетов влияют на Баланс, ДДС и Капитал.
- Периодичность колонок: по дням (период ≤ неделя), по неделям (<месяца), далее месяцы/кварталы/годы. Табличная форма (статьи в строках, периоды в колонках) — только формат, логика та же. Строки Баланса НЕ кликабельны (детализация через фильтр в Операциях). Фильтр по юрлицу. Экспорт в Excel. **При построении Баланса по проекту начальные остатки счетов не учитываются.**

### 5.4. Валютная переоценка (курс ЦБ РФ)
- **Правило 1 — по курсу на дату баланса (текущий):** денежные средства; неденежные дебиторская и кредиторская задолженности.
- **Правило 2 — по историческому курсу (на дату операции/начисления):** все доходы/расходы; весь капитал; запасы; основные средства; прочие внеоборотные/долгосрочные.
- Для статей Активов/Обязательств вариант выбирается (на дату операции / на дату баланса).
- Курсовые разницы оседают в Нераспределённой прибыли (удалить нельзя). Разница за период — в строке текущего периода; прошлые сворачиваются в текущий (в прошлых — ноль).
  - Тип 1 (от перемещений): `Сумма × (Курс на дату баланса − Курс на дату операции)`.
  - Тип 2 (от доходов/расходов): `Сумма × (Курс на дату баланса − Курс на дату начисления)`.

## 6. Учёт активов и обязательств

### 6.1. Общий приём ввода начальных остатков
Через служебную статью **«Корректировка»** (Капитал → Прочие статьи капитала), операцией **Начисление** без движения денег. Направление Дт/Кт зависит от того, актив это или обязательство. Сохраняет равенство Активы = Пассивы. ОПиУ и ДДС не затрагивает.

### 6.2. Запасы и себестоимость
- Покупка запасов = **Выплата** на статью Запасы (Актив). **НЕ расход.** ДДС: −деньги; Баланс: +Запасы.
- Продажа = **Поступление** на статью Продажа товаров (Доход), выручка по дате отгрузки.
- Списание себестоимости = **отдельное Начисление**: Дт Себестоимость (Расход) / Кт Запасы (Актив), сумма = закупочная цена отгруженного количества (не цена продажи). ОПиУ: расход; Баланс: −Запасы. Валовая прибыль = выручка − себестоимость.

### 6.3. Основные средства и амортизация
- Порог: >40 000 ₽ (бухучёт) / >100 000 ₽ (налог).
- Покупка ОС = Выплата на статью ОС (Актив). ДДС: инвестиционный поток.
- Амортизация = ежемесячное Начисление Дт Амортизация (Расход) / Кт ОС (Актив), линейно, сумма считается вручную, на последний день месяца. Неденежная — в ДДС не попадает. На Балансе ОС показывается нетто.
- Продажа выше балансовой: Поступление денег + Начисление Дт ОС / Кт Продажа оборудования (Доход) на превышение.
- Продажа ниже балансовой: Поступление + Начисление Дт Прочие расходы / Кт ОС (фиксация убытка).
- **Мультивалюта:** поле «Дата признания» фиксирует курс пересчёта по дате приобретения; без него счёт ОС не обнулится после полной амортизации (курсовой хвост).

### 6.4. Кредиты и полученные займы
- Статьи: «Кредиты» (Обязательства — тело), «Проценты по кредитам» (Расходы). Тело и проценты ВСЕГДА раздельно.
- Получение = Поступление на статью Кредиты (не доход, растёт обязательство).
- Погашение = Выплата с разбивкой: тело → Кредиты, проценты → Проценты по кредитам (расход).
- Начальный остаток (только тело, без процентов) = Начисление Дт Корректировка / Кт Кредиты.

### 6.5. Овердрафт
- «Овердрафт» = Обязательства → Другие краткосрочные; финансовая деятельность в ДДС.
- Выдача = Поступление на Овердрафт; погашение тела = Выплата на Овердрафт; проценты = Выплата на «Проценты по кредитам и займам».
- Начальный остаток = Начисление Дт Корректировка / Кт Овердрафт.

### 6.6. Лизинг
- Одновременно актив + обязательство. Статьи: «Лизинг» (Обязательства долгосрочные), предмет (ОС), Проценты.
- Постановка: либо Поступление+Выплата, либо Начисление Дт Предмет(ОС) / Кт Лизинг.
- Платёж = Выплата с разбивкой: тело → Лизинг, проценты → Проценты. К предмету применяется амортизация.

### 6.7. Выданные займы
- Актив (наша дебиторка): «Выданные займы до 1 года» (оборотные) / «от 1 года» (внеоборотные). Проценты → «Проценты по выданным займам» (Доходы → Прочие).
- Выдача = Выплата на статью займа (не расход). Возврат = Поступление с разбивкой: тело → займ, проценты → доход.

### 6.8. Дебиторка/кредиторка (комбинации дат)

| Сценарий | Тип операции | Подтверждена оплата | Подтверждено начисление | Результат |
|---|---|---|---|---|
| Предоплата клиента | Поступление | ✓ | ✗ | Неденежная кредиторка (мы должны отгрузить) |
| Предоплата поставщику | Выплата | ✓ | ✗ | Неденежная дебиторка (нам должны поставить) |
| Постоплата клиента | Поступление | ✗ | ✓ | Денежная дебиторка (нам должны деньги) |
| Постоплата поставщику | Выплата | ✗ | ✓ | Денежная кредиторка (мы должны деньги) |

- Иконки «Д» / «К». Контроль через фильтр по дате начисления (включается в Общих настройках), знак «!» = просрочено.
- **Частичная оплата:** создать операцию на полную сумму долга; каждый платёж — отдельная операция с ТОЙ ЖЕ датой начисления; сумму исходной операции уменьшать на оплаченное.

### 6.9. Факторинг
- Статья «Факторинг» (Актив). Признание продажи = Начисление Дт Факторинг / Кт Продажа (Доход). Получение денег нетто = Поступление против Факторинга. Комиссия = Начисление Дт Прочие расходы / Кт Факторинг (закрывает счёт в ноль).

### 6.10. Возвраты
- Клиенту: статья «Мы должны вернуть» (Обязательства). Сменить статью исходного Поступления на неё (или часть через «Разбить сумму»→Статья); при возврате — Выплата на эту статью. В ОПиУ не отражается.
- От поставщика: статья «Нам должны вернуть» (Активы). Аналогично — сменить статью исходной Выплаты, при получении — Поступление на статью.

### 6.11. Подотчёт
- Юрлицо «Подотчёт» + отдельный счёт на каждого сотрудника. Выдача = Перемещение на счёт сотрудника; трата = Выплата со счёта сотрудника (реальный контрагент/статья); возврат = Перемещение обратно. В расходы попадают только реальные траты.

### 6.12. Зарплата
- Метод 1 (прямой): Выплата на статью «Заработная плата» (Расход), дата начисления = месяц начисления; аванс — уменьшение исходной операции.
- Метод 2 (через обязательство): Начисление Дт Зарплата (Расход) / Кт «Долг перед сотрудником» (Обязательство); выплаты гасят долг. Баланс отслеживает остаток автоматически.

### 6.13. НДС (ставки РФ с 2025: 22%, 7%, 5%)
- Выделение из суммы с НДС: 22% → `×22/122`; 7% → `×7/107`; 5% → `×5/105`.
- Способ 1: одна статья «НДС» (Обязательства краткосрочные); каждую операцию разбивают на основу + НДС. Положительный остаток = долг бюджету.
- Способ 2: «Вх. НДС» (Активы оборотные) + «Исх. НДС» (Обязательства); зачёт фиктивной операцией.

### 6.14. Платежи третьим лицам / транзит / взаимозачёты
- «Платежи третьим лицам» (Обязательства): входящий платёж разбивают на доход + транзит; исходящий гасит обязательство. В ОПиУ — ноль, в ДДС — оба движения.
- Взаимозачёт: счёт «Взаимозачёт» в Активах (нетто-кредитор) или Обязательствах (нетто-должник); доход/расход проводятся Начислениями, остаток гасится деньгами.

### 6.15. Маркетплейсы (Ozon/WB/Яндекс)
- Счёт-актив для полной выручки + расходные статьи на каждую комиссию. Признание выручки = Начисление Дт счёт-актив / Кт Продажа (Доход); поступление денег нетто; комиссии = отдельные Начисления Дт Расход / Кт счёт-актив; себестоимость = Начисление Дт Себестоимость / Кт Запасы.

## 7. Сделки (Продажи / Закупки)

### 7.1. Поля сделки
`Название` (обяз.), `статус` (пользовательский, с цветным маркером), `контрагент` (клиент/поставщик), `юрлицо`, список товарных позиций, файлы. Вычисляемые: сумма сделки, прибыль, рентабельность (продажи), остаток денежного долга, остаток товарного долга.

### 7.2. Товарная позиция
`Количество` (дробное, до 3 знаков), `цена за единицу`, `скидка` (опц.). Сумма позиции = Кол-во × Цена − Скидка. Сумма сделки = Σ позиций.

### 7.3. Вкладки
- Продажа: Поступления, Отгрузки, Расходы, Счета.
- Закупка: Выплаты, Поставки.

### 7.4. Статусы (воронка)
Пользовательские, создаются вручную, цветные маркеры. Изменение: единично (в карточке) или массово (галочки → «Изменить»).

### 7.5. Расчёты
- **Поступило/Отгружено** (продажа) и **Выплачено/Поставлено** (закупка) — денежный и товарный учёт раздельно. Добавление операции автопересчитывает денежный остаток долга; добавление отгрузки/поставки — товарный остаток.
- **Прибыль сделки** = Сумма сделки − Себестоимость − Прочие расходы (доставка и т.п. с вкладки Расходы). Рентабельность = Прибыль / Сумма.
- Поля отгрузки/поставки: дата, юрлицо, клиент/поставщик, статьи отнесения. Могут быть частичными.

### 7.6. Связь с операциями/счетами/товарами
- Операция привязывается полем «Сделка»; это блокирует ручную дату начисления (берётся из отгрузки/поставки).
- Денежная операция может быть с любого счёта/любого контрагента.

### 7.7. Счета на оплату (инвойсы)
- Поля: № счёта (ручной/автонумерация), дата выставления, оплатить до (опц.), клиент, компания-отправитель (реквизиты), позиции (из сделки/вручную), ответственные (подписи PNG/JPG), логотип/печать (PNG/JPG ≤2 МБ, ≤800×700), формат ООО/ИП.
- Валюта счёта = валюте счёта отправителя. Неограниченное число счетов на сделку.
- **Создание счёта НЕ создаёт плановую операцию и НЕ влияет на отчёты** (чистый документ).
- **Автопривязка:** если в назначении входящего платежа указан № счёта — операция автоматически прикрепляется к сделке, **даже при переплате**.

## 8. Планирование

### 8.1. Два подхода
- **Бюджетирование (БДР/БДДС):** фиксированный план по статьям; цель — прибыль/исполнение бюджета; помесячные срезы.
- **Платёжный календарь:** динамический план (факт + прогноз); цель — кассовые разрывы; любая дата.

### 8.2. Общая модель бюджета
- Поля: Название, Период, Валюта; опц. Юрлицо, Проект/группа.
- Таблица: статьи (строки) × месяцы (столбцы). Ячейка: План, Факт, Выполнение %, Отклонение (абс), Отклонение %.
- Подстатьи: собственное значение (серым) + агрегат с подстатьями (синим).
- Формулы: `Выполнение % = Факт / План`; `Отклонение = Факт − План`; `Отклонение % = Отклонение / План`.
- Цвет: красный — перерасход по расходам/выплатам; зелёный — превышение по доходам/поступлениям. Критерий выполнения: поступления ≥ план И выплаты ≤ план.

### 8.3. Заполнение
- Автоимпорт «Подставить суммы»: если в прошлых периодах есть плановые операции, план = План + Факт, в столбец Факт — только реальные; при отказе — только факт.
- **Повторный импорт полностью перезаписывает план.** «Очистить колонку» необратимо. «Скопировать план…» на другие месяцы.

### 8.4. БДР (бюджет доходов и расходов)
- Только Доходы/Расходы. Метод: начисление (по дате начисления) или кассовый (по дате оплаты). План-факт для ОПиУ.

### 8.5. БДДС (бюджет движения денег)
- Все пять разделов статей (режим «Движение денег»). Кассовая логика. План-факт для ДДС.
- **Остаток на начало (1-й месяц)** = начальные остатки счетов (внесённые ранее) + фактические операции прошлого периода. Плановые операции прошлого периода НЕ включаются.
- Текущий/прошлые месяцы: остаток на начало = фактическому остатку на конец предыдущего.
- Будущие месяцы: плановый остаток на начало = плановому остатку на конец предыдущего.
- **Плановый остаток на конец** = плановый остаток на начало + общий плановый денежный поток (план + факт операции). Плановые перемещения НЕ учитываются.
- **Фактический остаток на конец** = факт. остаток на начало + фактические операции + начальные остатки, внесённые в этом месяце.
- Расхождения: перемещения между юрлицами, новые начальные остатки, курсовые разницы. **По проекту начальные остатки не учитываются.**

### 8.6. Платёжный календарь
- `План = фактические платежи + будущие платежи`. На любую дату — остаток на счетах.
- Режимы: «Движение денег» (все статьи) / «Доходы и расходы» (кассовый по дате оплаты / начисление по дате начисления).
- Периоды: дни/недели/месяцы/кварталы/годы. Разрезы: юрлица, счета, проекты, сделки.
- UI: знак «!» (просрочка), кнопка «На сегодня», клик по ячейке → формирующие операции, экспорт в Excel.

### 8.7. Кассовый разрыв
- Прогнозный остаток уходит в минус (при прибыльности). Индикация: шапка — красный (не покрыт другими счетами) / жёлтый (покрыт остатками других счетов); график «Остатки на счетах»; платёжный календарь («Движение денег» по дням).

### 8.8. План-факт в Показателях
- Текущий месяц = План + Факт; будущие — план; прошлые — факт. План отображается пунктиром.
- Источник плана проекта: бюджет проекта ИЛИ плановые операции (не смешивать).

### 8.9. Закрытие периода
- Режимы: до конкретной даты / по числу дней N (1–999, скользящая граница = сегодня − N).
- Проверка по ОБЕИМ датам (оплаты И начисления): любая в закрытом периоде → запрет редактирования/удаления/создания/групповых действий.
- **Исключения: импорт, интеграции, API НЕ блокируются.** Право: владелец и админ с правом настройки пользователей.

## 9. Проекты, контрагенты, товары/услуги, юрлица

### 9.1. Проекты
- Аналитический разрез; операция может быть частично отнесена к проекту (остаток «Не выбран» в проект не идёт); одну операцию можно делить между несколькими проектами через «Разбить сумму».
- Карточка: метрики Прибыль методом начисления / Прибыль кассовым методом / Денежный поток; рентабельность = Чистая прибыль / Выручка × 100% (производная, не хранится); список привязанных операций.
- Список: вид «По группам», сортировка, фильтр по датам, видимость архивных, итоги, экспорт в Excel.
- Группа = сумма показателей входящих проектов за период/метод.
- Проект не вводит активов/обязательств → на Баланс напрямую не влияет.

### 9.2. Контрагенты (дебиторка/кредиторка)
- См. §1.3, §6.8. Сальдо взаиморасчётов по контрагенту/группе из разрыва дат оплаты/начисления. Сотрудники = контрагенты.

### 9.3. Товары/Услуги (НДС)
- Справочник номенклатуры (импортируемый). Позиции используются в сделках и счетах. Количество дробное (3 знака). НДС учитывается разбиением операции на основу + статью НДС (§6.13).

### 9.4. Юрлица
- См. §1.1. Ограничение: нельзя жёстко закрепить статьи/проекты за конкретным юрлицом — разделение направлений делается через Проекты; фильтр по юрлицу — в отчётах. Несколько бизнесов в одном аккаунте = проекты + группы.

## 10. Импорт данных, фильтры, прочие нюансы

### 10.1. Источники ввода
Ручной ввод; банковская выписка (1С `.txt` UTF-8, для РБ — XML UTF-8, ≤50 МБ, файл нельзя редактировать вручную); Excel по фиксированному шаблону (обяз. поля: дата, сумма, тип, счёт, контрагент); интеграции (банки/карты, 1С, маркетплейсы, эквайринг, AmoCRM); мобильное приложение; Telegram-бот (обяз. Сумма+Счёт, через запятую, знак суммы = направление, новые сущности создавать нельзя). Если выписка не покрывает период — автосоздание операции начального остатка.

### 10.2. Сверка при импорте (статусы)
- **Новая** (загрузить), **Существует** (совпадает), **Изменена** (расхождение по дате/сумме/счёту — влияет на остаток, править под выписку), **Нет в выписке** (лишняя ручная — удалить/снять подтверждение).
- Сопоставление план/факт: люфт дат ≤45 дней + точное совпадение суммы (до копеек) + счёт + контрагент.
- Платёж между своими юрлицами/счетами → всегда Перемещение. «Это не Я» / преобразование контрагента в юрлицо переключают тип операции.

### 10.3. Правила импорта
- Условие (счёт/контрагент/направление/сумма/текст назначения) → действия: назначить статью/проект, заменить контрагента, разбить на части (% долями), назначить/распределить дату начисления (1–12 мес ∝ дням).
- **Только первое подходящее правило применяется** (по порядку списка сверху вниз), не суммируются. Правило важнее статьи из карточки контрагента. Применение необратимо. Не применяется к уже разбитым вручную операциям. «Применить к существующим» с предпросмотром.
- Депозиты/овернайт: схлопывание счетов по первым 3 цифрам номера + юрлицо.

### 10.4. Фильтры
- **Общие (левые):** период (диапазон дат), счёт, юрлицо, контрагент, проект/группа, сделка.
- **Быстрые фильтры** (сохраняемые наборы) в ОПиУ/ДДС и Операциях.
- **Плавающие даты:** Вчера / Эта неделя / Прошлая неделя / Этот месяц / Прошлый месяц / Просроченные — пересчитываются при каждом применении.
- Колонка «Тренд» (день/неделя/месяц, сравнение с прошлым периодом / аналогичным прошлого года, drill-down).

### 10.5. Общие настройки
- Валюта аккаунта по умолчанию; классификация расходов (прямые/косвенные | постоянные/переменные | нет); название бизнеса; подстановка текущей даты при копировании; показ поля «Дата начисления»; обязательность «Назначение платежа»; отображение копеек (только вывод — в расчётах копейки ВСЕГДА); видимость прошлых кассовых разрывов; фильтр по дате начисления; показ номеров платёжных поручений.

### 10.6. Права, уведомления, демо
- Гибкие права по разделам/действиям; владелец/админ. Email-уведомления о денежном потоке (понедельник 10:00–12:00). Демо-бизнес. Центр уведомлений.

### 10.7. Сквозные крайние случаи (критично для клона)
- Дата начального остатка строго раньше первой операции.
- Если дата начисления пуста → = дате оплаты.
- Перемещения и транзитные платежи/взаимозачёты не должны попадать в доходы/расходы ОПиУ.
- Начисления и амортизация не попадают в ДДС.
- Покупка запасов/ОС — не расход в момент оплаты; расход рождается списанием себестоимости/амортизацией.
- Привязка к сделке блокирует дату начисления.
- Знак ноги Начисления = инверсия по типу статьи (актив/пассив), не просто Дт/Кт.
- Undo нет; групповые действия и правила импорта необратимы; «Отменить разбиение» теряет данные частей; групповые действия недоступны для разбитых операций.
- Копейки всегда в расчётах. Все валютные курсы — ЦБ РФ; курсовые разницы оседают в Нераспределённой прибыли (удалить нельзя).
- Баланс накопительный с начала учёта (не за период); по проекту начальные остатки счетов игнорируются.
- Закрытие периода проверяет обе даты, но не блокирует импорт/API.
- «Итого» по счёту в Операциях не включает начальный остаток.
- Деление займов «до 1 года» (оборотные/краткосрочные, опер. поток) vs «от 1 года» (внеоборотные/долгосрочные, инвест/фин. поток) влияет на все три отчёта.
- Нераспределённая прибыль = чистая прибыль − дивиденды + курсовые разницы (не просто чистая прибыль).
