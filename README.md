# Управленческий учёт (аналог ПланФакта)

Самостоятельное веб-приложение управленческого учёта для малого и среднего бизнеса —
аналог planfact.io. Отчёты **ОПиУ** (P&L), **ДДС** (cash flow) и **Баланс** (Активы = Пассивы),
платёжный календарь, сделки, бюджеты (БДР/БДДС), импорт/экспорт и многое другое.

- **Backend:** FastAPI + PostgreSQL (async SQLAlchemy 2.0)
- **Frontend:** React + Vite + Tailwind + recharts
- **Запуск:** одна команда через Docker Compose

> Бэкенд сам отдаёт собранный фронтенд — отдельный веб-сервер для фронта не нужен.

## Возможности

- **Операции**: поступления, выплаты, перемещения (в т.ч. «парные» — деньги в пути с разными
  датами/счетами), начисления (двойная запись), отгрузки/поставки; разбиение суммы на части;
  фильтры; **массовое выделение и групповое изменение/удаление** с итогами по выбранным.
- **Отчёты**: ОПиУ (метод начисления и кассовый, разрезы по статьям/проектам/сделкам,
  EBITDA/EBIT/EBT), ДДС по видам деятельности, Баланс (накопительный, дебиторка/кредиторка,
  деньги в пути), платёжный календарь, дашборд с Парето 20/80. Экспорт отчётов и операций в Excel.
- **Справочники**: статьи (дерево с подкатегориями), счета, контрагенты, проекты, **юрлица**,
  товары/услуги с НДС.
- **Сделки** с расчётами (поступило/выплачено/долг/прибыль/рентабельность), **бюджеты** (БДР/БДДС,
  план-факт).
- **Импорт**: мастер с сопоставлением колонок (Excel/CSV), банковские выписки **1С
  (1CClientBankExchange)** и CSV-выписки банков (Тинькофф — одна колонка со знаком; Сбер и др. —
  отдельные колонки приход/расход), сохранённые правила импорта.
- **Дисциплина учёта**: закрытие периода, повторяющиеся операции, мультивалютность, фильтр по юрлицу.

## Быстрый старт (Docker)

Нужен сервер или компьютер с установленными **Docker** и **docker compose**.

```bash
git clone <URL-этого-репозитория> planfact && cd planfact
cp .env.example .env
# отредактируйте .env: задайте POSTGRES_PASSWORD, JWT_SECRET (openssl rand -hex 32),
# ADMIN_EMAIL/ADMIN_PASSWORD
docker compose up -d --build
```

Через минуту приложение доступно на **http://АДРЕС-СЕРВЕРА:8095**
(порт задаётся переменной `APP_PORT` в `.env`).

Войдите под `ADMIN_EMAIL` / `ADMIN_PASSWORD` из `.env`. Если `SEED_DEMO=1`, внутри уже будет
демокомпания с примерами операций и сходящимся балансом — удобно для знакомства.

Подробное развёртывание на чистом сервере (установка Docker, домен + HTTPS, бэкапы,
обновления) — в [DEPLOY.md](DEPLOY.md).

## Локальная разработка (без Docker)

Backend (Python 3.11+):
```bash
cd backend
python -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env            # укажите DATABASE_URL к локальному PostgreSQL и JWT_SECRET
python -m app.bootstrap --demo  # схема + админ (ADMIN_EMAIL/ADMIN_PASSWORD) + демоданные
uvicorn app.main:app --reload --port 8000
```

Frontend (Node 18+):
```bash
cd frontend
npm install
npm run dev      # http://localhost:5173, /api проксируется на :8000
```

## Структура

```
backend/    FastAPI, SQLAlchemy-модели, сервисы отчётов, bootstrap (инициализация БД)
frontend/   React + Vite + Tailwind
docs/       Спецификация поведения учёта и карта интерфейса
deploy/     Пример systemd-юнита для ручного развёртывания (не нужен при Docker)
```

Документы в `docs/` описывают правила учёта (две даты: оплата→ДДС, начисление→ОПиУ;
перемещения и начисления не в ДДС; покупка актива — не расход; накопительный Баланс и т.д.).
